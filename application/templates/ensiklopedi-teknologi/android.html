Android

Jenis: Platform Sistem Operasi
Pembuat: Google
Macam versi: Android 1.5 (Cupcak), Android 2.2 (FroYo), dlsb.
Tools yang dibutuhkan:

  • Android SDK (Software Development Kit)
  • Android Emulator
Perangkat yang perlu diinstal:
  • Android SDK (Software Development Kit)
    • SDK Starter Package
    • SDK Components
  • Android Emulator
  • Android Virtual Device (AVD) sebagai emulator android
  • JDK (Java Development Kit)
  • Sebuah IDE (Integrated Development Environment), misal: Eclipse
    • Eclipse Plugin: ADT (Android Development Toolkit)
Langkah:
  • Instal Eclipse IDE for Java Developers
  • Instal Android SDK
  • Instal Plugin ADT pada Eclipse
    • Pada Eclipse, Help> Install New Software> Add Site
    • Situs ADT: (https/http):dl-ssl.google.com/android/eclipse/
  • Membuat program sederhana
    • Pada Eclipse, File> New> Project> Android> Android Project
    • Isikan project name, build target, app name, dlsb
    • Setelah dibuat, Project> Clean
  • Optional: Gunakan emulator/ ponsel sungguhan
    • Gunakan emulator: instal AVD
      • Pada Eclipse, Window> AVD Manager
      • Isikan name, target, dlsb kemudian klik Create AVD
  • Optional: Gunakan ponsel sungguhan
    • instal Android USB device driver
    • Hubungkan ponsel menggunakan kabel aslinya
  • Menjalankan program Android
    • Klik kanan pada ProjectName> Run As> Android Application
Izin:
  • INTERNET (untuk akses internet)
  • READ_CONTACTS (membaca, tanpa mengubah, data kontak pengguna)
  • WRITE_CONTACTS (mengubah, tanpa membaca, data kontak pengguna)
  • RECEIVE_SMS ()
  • ACCESS_COARSE_LOCATION (menggunakan layanan lokasi yang belum jelas seperti cell towers)
  • ACCESS_FINE_LOCATION (menggunakan layanan lokasi yang jelas seperti GPS)

    • Contoh Penerapannya:
    <manifest xmlns:android="http://schemas.android.com/apk/res/android" package="com.google.android.app.myapp" > <uses-permission android:name="android.permission.RECEIVE_SMS" /> </manifest>
Layout Android:
  • FrameLayout: menyusun komponen-komponen sehingga mereka memulai dari ujung atas kiri pada layar
  • LinearLayout: menyusun komponen-komponen pada satu kolom atau baris
  • RelativeLayout: menyusun komponen-komponen berdasarkan hubungannya satu sama lain atau dengan induknya
  • TableLayout: menyusun komponen-komponen pada baris-baris dan kolom-kolom, mirip dengan tabel pada HTML

    • Parameter: xmlns:android="http://schemas.android.com/apk/res/android"
    Untuk layar horisontal/landscape, maka android:orientation="horizontal"
    Untuk layar vertikal/portrait, maka android:orientation="vertical"
    Contoh Penerapannya:
    <?xml version="1.0" encoding="utf-8"?> <LinearLayout xmlns:android="http://schemas.android.com/apk/res/android" android:orientation="vertical" android:layout_width="fill_parent" android:layout_height="fill_parent"> <TextView android:layout_width="fill_parent" android:layout_height="wrap_content" android:text="@string/main_title" /> <Button android:layout_width="fill_parent" android:layout_height="wrap_content" android:text="@string/continue_label" /> <Button android:layout_width="fill_parent" android:layout_height="wrap_content" android:text="@string/new_game_label" /> <Button android:layout_width="fill_parent" android:layout_height="wrap_content" android:text="@string/about_label" /> <Button android:layout_width="fill_parent" android:layout_height="wrap_content" android:text="@string/exit_label" /> </LinearLayout>
Pixel:
  • px (pixels): titik-titik pada screen
  • in (inches): ukuran seperti yang ada pada penggaris
  • mm (milimeters): ukuran seperti yang ada pada penggaris
  • pt (points): 1/72 inch
  • dp/dip (density-indepenent pixel): abstraksi satuan berdasarkan besaran kepadatan pada layar. Misal pada layar dengan 160 dpi (dots per inch), 1 dp = 1px
  • sp (scale-independent pixels): mirip dengan dp tetapi diskala kembali berdasarkan font size yang diset pengguna

    • Rekomendasi, gunakan:
  • sp: untuk ukuran teks
  • dp/dip: untuk yang lain

Perlu Diketahui

Aplikasi android yang dirancang untuk versi android yang lebih lama tidak bisa digunakan untuk android dengan versi yang lebih baru. Namun sebaliknya, aplikasi android untuk versi android yang lebih baru bisa digunakan untuk android dengan versi yang lebih lama.
